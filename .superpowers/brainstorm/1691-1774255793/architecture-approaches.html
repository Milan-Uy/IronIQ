Architecture Approach

How should the AI coach, workout engine, and UI work together?

A

Chat-Centric (Recommended)

The AI chatbot is the primary interface. Users interact conversationally to create, modify, and review workouts. The chat produces structured JSON that feeds into workout views, tracking, and comparison screens.

Pros

  • Natural coaching experience
  • Single interaction model
  • AI context stays coherent

Cons

  • Power users may want direct editing
  • Chat can feel slow for quick changes
B

Dashboard + Chat Sidebar

Traditional fitness app UI (dashboard, workout cards, tracking forms) with an AI chat sidebar you can open anytime. Users can edit workouts directly OR ask the AI. Chat modifications update the same data.

Pros

  • Familiar app feel
  • Fast direct editing + AI help
  • Better for quick logging

Cons

  • More UI to build
  • Two interaction paradigms to maintain
C

Hybrid Tabs

Tab-based layout: Coach (chat), My Program (structured view), Track (logging), Insights (analytics). The chat tab is the AI coach, other tabs are traditional UI. Actions in chat reflect across tabs in real-time.

Pros

  • Clear separation of concerns
  • Each feature gets dedicated space
  • Scales well as features grow

Cons

  • Most to build upfront
  • Context switching between tabs
